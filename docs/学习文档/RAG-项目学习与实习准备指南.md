# RAG 项目学习与暑期实习准备指南

> 版本：v0.1  
> 适用对象：当前对 RAG 几乎零基础，但希望在 1 个月内吃透自己的 RAG 项目、完成基础项目表达，并具备暑期实习面试应对能力。  
> 文档定位：长期维护的项目内学习文档。既服务于我自己的学习复盘，也作为 GitHub Copilot / ChatGPT / Gemini 等 AI 工具的上下文约束文档。  
> 推荐放置位置：`docs/学习文档/RAG-项目学习与实习准备指南.md`

---

## 0. 这份文档是用来做什么的

这份文档解决三个问题：

1. **让我始终知道自己现在应该学什么**，避免被大量概念和 AI 解释带偏。
2. **让 AI 工具围绕我的真实目标进行教学**，而不是泛泛讲理论。
3. **让项目学习、项目表达、实习准备三条线同步推进**，最终能够把 RAG 项目讲清楚、讲可信。

这不是一份“学完就丢”的临时清单，而是一份可以随着项目进展不断补充的工作文档。

## 1. 当前总目标

### 1.1 一个月目标

在 1 个月内，达到以下状态：

- 能完整讲清楚我的 RAG 项目在做什么。
- 能说清楚核心调用链和主要模块职责。
- 能回答基础 RAG 概念问题。
- 能回答基础 Java 后端与工程实现追问。
- 能围绕该项目完成暑期实习中的基础项目面。
- 至少亲自补一个小改动或一个小优化点，让项目表达更有底气。

### 1.2 明确边界

当前阶段的目标不是：

- 把 RAG 理论研究到很深。
- 把向量检索底层算法彻底啃透。
- 把所有八股全部刷完。
- 把所有技术点都讲成专家级。

当前阶段更现实的目标是：

- **先吃透自己的项目，再补足足够应对基础面试的理论。**

## 2. 学习原则

### 2.1 项目优先

学习顺序必须以项目为核心，不以抽象概念为核心。

优先级：

1. 看懂项目主链路
2. 看懂关键模块
3. 反推相关 RAG 概念
4. 再补 Java 后端知识点
5. 最后做表达和模拟面试

### 2.2 输出优先于“感觉懂了”

AI 的解释只能辅助理解，不能代替真正掌握。

每天都必须至少有一种输出：

- 文字总结
- 手画/电子流程图
- 录音讲解
- 问答卡片
- 代码批注
- 一个小改动或一个小测试

### 2.3 只学与目标强相关的内容

遇到新概念时先判断：

- 它是否直接出现在项目里？
- 它是否是面试高频追问点？
- 它是否能帮助我解释项目设计？

如果都不是，先降优先级。

### 2.4 让 AI 当导师，不让 AI 代替我思考

AI 可以负责：

- 拆模块
- 解释代码
- 出题追问
- 检查回答
- 帮我找盲点

我自己必须负责：

- 总结
- 复述
- 组织语言
- 记录理解
- 校验是否真的会讲

## 3. 一个月学习主线

整个学习过程分成三条线并行推进：

### 3.1 主线 A：吃透 RAG 项目

这是最核心的主线。

目标：

- 看懂项目主链路
- 看懂主要模块职责
- 知道为什么这么设计
- 能用自己的话讲出来

### 3.2 主线 B：补和项目强相关的 Java 后端基础

目标：

- 不泛学
- 只补项目里真正会被问到的基础

重点围绕：

- Spring Boot
- IOC / AOP
- MyBatis / MyBatis-Plus
- Redis
- JWT
- SSE / WebClient
- 异步任务
- 限流 / 幂等
- 基础 SQL 与索引

### 3.3 主线 C：项目表达与面试准备

目标：

- 项目介绍讲顺
- 高频追问答顺
- 简历话术和真实实现对齐

## 4. 学习路线图

---

### 阶段 1：先看懂项目是怎么跑起来的（第 1 周）

目标：建立全局视角。

#### 这一阶段必须完成的结果

- 画出项目完整流程图
- 知道每个模块大致做什么
- 能讲出 3 分钟项目介绍
- 建立自己的 RAG 术语表

#### 这一阶段重点任务

1. 阅读 README、模块结构、配置文件
2. 从入口往下追主链路
3. 梳理问答链路与文档处理链路
4. 记录每个模块的职责
5. 建立“项目总图 + 模块图 + 调用链图”

#### 本阶段核心问题

- 这个项目到底解决什么问题？
- 用户如何从上传文档走到问答？
- 文档是如何被处理并进入向量库的？
- 问题是如何变成答案的？
- 哪些模块属于业务层，哪些属于基础设施层？

---

### 阶段 2：补 RAG 核心概念，但只围绕项目来学（第 2 周）

目标：知道项目设计背后的基本原理。

#### 这一阶段必须搞懂的概念

1. 什么是 RAG
2. 什么是 Embedding
3. 为什么需要向量库
4. 什么是 Chunk / Chunk Overlap
5. 什么是 Recall / TopK / Score
6. 什么是 Rerank
7. PromptBuilder 在做什么
8. 什么是 Hallucination
9. Citation 为什么有价值
10. 生成模型和 Embedding 模型的区别

#### 本阶段核心问题

- 为什么不能把整篇文档直接扔给大模型？
- 为什么分块大小会影响效果？
- 为什么相似检索依赖 embedding？
- 为什么要控制 prompt 的上下文长度？
- 为什么 RAG 不能完全避免幻觉？
- 为什么引用来源是一个重要能力？

---

### 阶段 3：把项目真正变成“我自己的项目”（第 3 周）

目标：从“看懂”走向“能讲、能改、能复盘”。

#### 这一阶段必须完成的结果

- 至少做一个小改动或小优化
- 写一份项目复盘
- 准备 20 个项目追问题
- 开始模拟面试

#### 可选小改动方向

- 给问答链路增加更清晰的日志/耗时记录
- 优化 PromptBuilder 输出格式
- 增加一个简单的调试输出
- 给缓存命中与未命中增加更直观的说明
- 做一个简单的评测记录
- 增加一个小测试或一个错误处理改进

#### 本阶段核心问题

- 如果面试官问“你做了什么”，我能讲出哪些亲手做的点？
- 如果面试官问“你的优化点是什么”，我能回答什么？
- 如果面试官问“你这个项目还有什么不足”，我是否能坦诚回答？

---

### 阶段 4：面试冲刺与实习准备（第 4 周）

目标：把学习成果转化成表达能力。

#### 这一阶段必须完成的结果

- 准备 30 秒 / 2 分钟 / 5 分钟三版项目介绍
- 完成至少 3 次项目模拟面
- 完成至少 2 次 Java 基础模拟面
- 完成一次简历整体串讲

#### 本阶段核心问题

- 我能否在 30 秒内把项目讲明白？
- 我能否在 2 分钟内把项目的亮点说清楚？
- 我能否在被连续追问时仍然不乱？

## 5. 我真正需要学会的内容清单

这一部分不是“技术大全”，而是当前阶段必须吃透的最低集合。

### 5.1 RAG 项目层面必须掌握的内容

#### A. 全链路视角

我必须能不看代码讲清楚以下链路：

用户提问  
→ 问题向量化  
→ 向量检索相关 chunk  
→ 结果过滤 / 重排  
→ 组装 prompt  
→ 调用大模型生成答案  
→ 返回答案与引用

#### B. 文档处理链路

我必须知道：

- 文档如何上传
- 为什么采用异步处理
- 文档如何解析
- 为什么要分块
- 为什么会有 chunk overlap
- 向量是如何生成的
- 向量和元数据如何入库

#### C. 问答链路

我必须知道：

- 问题是如何进入问答服务的
- 检索是怎么做的
- topK / score 是什么
- PromptBuilder 在做什么
- LLM 是如何被调用的
- 为什么支持流式输出
- 引用来源是如何构造的

#### D. 工程能力

我必须知道：

- Redis 缓存缓存了什么
- 为什么需要限流
- 为什么需要幂等
- 为什么需要重试
- 多 provider / 多 vector store 的抽象意义是什么

## 6. 项目学习优先顺序

建议按照以下顺序逆向学习：

1. `README` 和整体结构
2. `QAController`
3. `RAGServiceImpl`
4. `QueryEngineImpl`
5. `PromptBuilder`
6. `AnswerGeneratorImpl`
7. `DocumentIndexingServiceImpl`
8. `DocumentProcessorImpl`
9. `EmbeddingServiceImpl`
10. `VectorStoreConfig`
11. 限流 / 幂等 / 缓存相关模块
12. 前端问答展示与接口对接

### 为什么按这个顺序

因为这个顺序符合：

- 先抓用户主路径
- 再下潜核心服务
- 再补基础设施

避免一开始钻到配置和底层实现里迷路。

## 7. 每日学习工作流

每天学习建议采用固定模板。

### 7.1 推荐时长

如果时间允许，推荐每天 3～5 小时。

### 7.2 每日模板

#### 第一段：读代码（60～90 分钟）

只看一个模块，不要贪多。

输出：

- 这个模块是干什么的
- 输入是什么
- 输出是什么
- 依赖谁
- 核心方法是什么
- 我当前看不懂什么

#### 第二段：AI 辅助解释（45～60 分钟）

把刚看的模块交给 Copilot / ChatGPT / Gemini 解释。

目标：

- 不让 AI 泛讲
- 只围绕当前文件和当前问题展开

#### 第三段：自己总结（20～30 分钟）

必须自己写总结。

建议格式：

- 模块职责
- 调用链位置
- 关键类 / 方法
- 面试可能怎么问
- 我还不懂的点

#### 第四段：讲出来（10～15 分钟）

对着屏幕讲，或者录音。

标准：

- 不能只会看
- 要能说出来

#### 第五段：小练习（30～45 分钟）

可选：

- 打断点
- 跑接口
- 改一行代码
- 加日志
- 写一个小测试
- 调整一个配置

## 9. GitHub Copilot 的使用方式

我已经习惯用 `copilot-instructions.md` 约束 Copilot，因此需要让 Copilot 在本项目中承担“导师 + 追问者 + 检查器”的角色。

### 9.1 Copilot 应扮演的角色

Copilot 在本项目学习中应承担以下角色：

- 项目代码导师
- 模块拆解助手
- 问答追问器
- 学习检查器
- 面试模拟器

Copilot 不应该只做：

- 一次性给出大量解释
- 大段泛泛理论输出
- 直接替我得出结论

### 9.2 Copilot 的教学原则

我希望 Copilot：

1. 先讲模块职责，再讲实现细节。
2. 优先基于项目真实代码讲解，不脱离仓库泛讲。
3. 每次只推进一小步。
4. 解释后要给我出检查问题。
5. 在我回答后继续追问，帮助我暴露盲点。
6. 对我回答中的错误和模糊点进行纠正。
7. 引导我用自己的话复述，而不是只看解释。

### 9.3 建议写入 `copilot-instructions.md` 的约束方向

可加入类似要求：

- 你在本仓库中不仅是代码助手，也是项目学习导师。
- 当我要求你讲解代码时，优先基于当前仓库的真实文件、类、方法与调用链解释。
- 不要一次性输出过多内容，应分步骤推进。
- 优先回答“这个模块是干什么的、在项目中的位置、为什么这样设计”。
- 每次讲完后请给我 3～5 个检查问题，检验我是否真正理解。
- 当我回答问题时，请指出其中不准确、空泛、缺少工程细节的地方，并要求我重说。
- 当我要求你模拟面试时，请像真实面试官一样连续追问，不要轻易放过模糊回答。
- 当我要求你辅助学习某个模块时，请输出：模块职责、调用链位置、关键方法、典型追问、我的学习重点。

## 10. AI 工具协作分工建议

### 10.1 GitHub Copilot

适合做：

- 当前代码文件的局部讲解
- 类 / 方法 / 调用链拆解
- 直接在仓库上下文里教学
- 基于当前代码出题
- 帮我写注释、日志、测试、微小改动

### 10.2 ChatGPT / GPT-5.4 Thinking

适合做：

- 帮我梳理学习路线
- 帮我校正项目表达
- 帮我做面试追问
- 帮我把一个模块上升到“如何讲给面试官听”
- 帮我发现简历和真实实现之间的风险点

### 10.3 Gemini

适合做：

- 换一种讲法解释概念
- 用类比帮助我理解抽象点
- 帮我重述成更直观的语言

### 10.4 使用原则

同一个问题不要无限切工具。

推荐方式：

- **先自己看代码**
- 再让 Copilot 解释当前文件
- 再让 ChatGPT 帮我提升表达和面试答案
- 最后必要时让 Gemini 换一种更直观的说法

## 11. 建议使用的高质量提问模板

以下模板是为了让 AI 真正服务当前目标。

### 11.1 模块学习模板

```text
请你把自己当作我的项目导师，基于当前仓库中的这个文件带我逆向学习。
要求按以下结构输出：
1. 这个模块的职责
2. 它在整个项目调用链中的位置
3. 关键类/方法的作用
4. 为什么要这么设计
5. 面试官可能会怎么追问
6. 最后给我 5 个检查问题
不要一次性讲太泛，优先结合当前代码。
```

### 11.2 回答检查模板

```text
下面是我对这个模块的理解，请你像严格的导师一样检查。
请指出：
1. 哪些地方说对了
2. 哪些地方过于空泛
3. 哪些地方不准确
4. 哪些关键点我漏掉了
然后要求我重新组织答案。
```

### 11.3 面试追问模板

```text
你现在是 Java 后端实习面试官。
请基于我的 RAG 项目连续追问我 10 个问题。
要求：
- 问题从浅到深
- 每次等我回答后再继续追问
- 如果我的回答很空泛，请你继续压问细节
- 重点关注项目真实性、设计理由、工程实现和优化点
```

### 11.4 概念理解模板

```text
请不要脱离我的项目泛讲概念。
请结合我的 RAG 项目解释“[概念名]”。
要求输出：
1. 它是什么
2. 在我的项目里对应哪里
3. 为什么需要它
4. 面试官可能会怎么问
5. 我应该如何用口语化方式回答
```

### 11.5 小改动教学模板

```text
我想通过一个小改动真正掌握这个模块。
请你基于当前代码，给我 3 个难度适中的小改动建议。
每个建议请说明：
1. 改动目标
2. 涉及哪些文件
3. 我能学到什么
4. 面试时可以怎么讲
优先选择对当前项目有价值、我一两天内能完成的改动。
```

- 30 秒讲项目不卡壳

- 被追问时能继续说细节
